指针小结

2023-02-20
  • C语言

自从2月8号以来已经过去了12天，咕了三次本应该有的学习小结，而真正能整理出来的笔记却只有大约半天的学习量。非常惭愧，还是要端正态度。

指针使用

scanf("%d", &i); //取地址运算：&运算符取得变量的地址
//指针变量就是记录地址的变量
int* p = &i;

int* p;
int *p;
//以上两种声明方式等价，意为声明了一个int类型的指针。

int k = *p; //*是一个单目运算符，用来访问指针所指向的地址上的变量
*p = k + 1;
  • 变量的值是内存的地址
    • 普通变量的值是实际的值
    • 指针变量的值是具有实际值的变量的地址

指针运用场景

1. 函数返回多个值，某些值只能通过指针返回

传入的参数实际上是需要保存带回结果的变量
例：交换两个变量的值

void swap(int *pa, int *pb) {
  int t = *pa;
  *pa = *pb;
  *pb = t;
}

2.函数返回运算的状态，结果通过指针返回

返回函数的运算状态，通常的做法是让函数返回特殊的不属于有效范围的值，如0，-1
但当任何数值都是有效的可能结果时，就得分开返回（状态通过函数返回值返回，实际值通过指针返回）
ps：在C++和Java采用了异常机制来处理这个问题
例：一个做除法的函数

int divide(int a, int b, int *result) {
  int ret = 1;
  if (b == 0) {
    ret = 0;
  }
  else {
    *result = a / b;
  }
  return ret;
}

指针和数组

数组作为参数传入函数时，实际上传入的是一个指向数组的指针。

  • 数组变量是特殊的指针
    • int a[10]; int *p = a; // 不用&取地址
    • 但是数组的单元表达的是变量，需要用&取地址。
    • a == &a[0]
  • []运算符和*运算符既可以对数组做运算，也可以对指针做运算
    • p[0] <===> a[0]
    • *a = 25 <===> a[0] = 25;
  • 数组变量是const的指针，所以不能被赋值
    • int a[] <===> int * const a = ….

指针和const

// （1）
int *const q = &i;
*q = 26; // ok
q ++; // error!
// 表示一旦得到了某个变量的地址，指针就不能再指向其他变量 【指针q是const】

// (2)
const int *p = &i;
// 表示不能通过这个指针p去修改变量i（不会使变量i成为const）【指针p所指的变量是const】
*P = 26; // error! (*p)是const
i = 26; // ok
p = &j // ok
//通过判断const在*前还是后来做区分
const int *p1 = &i; // （2）
int const* p2 = &i; // （2）
int *const p3 = &i; // （1）

使用

  1. 当要传递的参数类型比地址大的时候，既能用比较少的字节数传递值给参数(特别是在结构体中），又能避免函数对外面的变量的修改
void f(const int* x);
  1. 保护数组值，因为把数组传入函数时传递的时地址，所以那个函数内部可以修改数组的值，为了避免数组不被函数修改，可以设置参数为const
int sum(const int a[],int length);

指针运算

//给一个指针加一表示要让指针指向下一个变量
int a[10];
int *p = a;
*(p + 1) == a[1];
// 如果指针不是指向一片连续分配的空间，如数组，则这种运算没用意义
*p ++;
//取出p所指的数据，然后把p移到下一个位置（*的优先级小于++）

0地址

内存中有0地址，但通常指针不应该具有0值。

  • 因此可以用0地址表示特殊的事情：
    • 返回的指针时无效的
    • 指针没有被真正初始化
  • NULL时一个预定定义的符号，用来表示0地址
  • 使用NULL而不是0来表示0地址！

指针类型转换

  • void表示不知道指向什么类型的指针
int *p = &i; void *q - (void*)p; //指针的类型转换
//这里并没有改变p所指的变量的类型，而是改变了p指向的类型
//p指向了int -> p指向了void

动态内存分配

输入数据时，先给你数据个数n，再给你n个数据，要求你存储这n个数据，该怎么做？

int *a = (int*) malloc(n * sizeof(int));
for(int i = 0; i < n; i ++) {
  scanf("%d", &a[i]);
}
free(a);
// C99开始可以用变量确定数组的大小,C99之前只能用以上方式
  • 向malloc申请的空间的大小是以字节为单位的
#include <stdlib.h>
void* malloc(size_t size);
  • 返回结果时void*。需要将类型转换为自己需要的类型
  • 如果没空间，申请失败了会返回0，即NULL
  • free() 申请来的空间得“还”
    • 只能还申请来的空间的首地址
  • 指针定义时最好用NULL做一下初始化，以免它指向奇怪的地方
    • 因此free(NULL)是被允许的
free(NULL); //ok
void *p = NULL; // 
free(p) //ok

常见问题

  • malloc后没free -> 长时间运行内存逐渐下降
  • free过了再free
  • 地址变了，直接去free
